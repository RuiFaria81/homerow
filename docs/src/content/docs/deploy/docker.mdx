title: Deploy with Docker / Podman
description: Deploy without needing a local Nix installation.

If you don't want to install Nix locally, this is the simplest path.

:::note
For most users, the recommended method is [Fork-and-Deploy (GitHub Actions)](/deploy/github-actions/).
Use this local Docker/Podman path when you want deployment to run from your own machine.
:::

<Steps>
1. **Clone and enter the repository:**
   ```bash
   git clone <repo-url>
   cd homerow
   ```

2. **Create `config.env`** in the repo root. See [Configuration](/getting-started/configuration/) for the full reference and the complete example.

3. **Configure your SSH key** — reuse an existing key or generate a dedicated one:
   ```bash
   ssh-keygen -t ed25519 -f ~/.ssh/homerow_ed25519 -N ""
   ```

4. **Make sure Docker or Podman is installed.**

5. **Run deploy** using key path:
   ```bash
   SSH_PRIVATE_KEY_PATH=<path/to/private_key> ./hrow docker deploy
   ```
</Steps>

:::note
Deploy always needs an SSH key. For this local method, set `SSH_PRIVATE_KEY_PATH`.
:::

On the first run, the deploy image is built automatically before starting the full deployment.
