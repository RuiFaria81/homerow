Use this method if you prefer running deploy from GitHub instead of your local machine.

<Steps>
1. **Fork the repository** on GitHub.

2. **Set repository secrets** in your fork with the same variable names from the [Configuration](/getting-started/configuration/) reference, including `SSH_PRIVATE_KEY`.

3. **Run the workflow** — go to Actions and run `Deploy Mail Server`.
</Steps>

### Optional (Recommended): easier secret setup

Instead of manually adding each secret in the GitHub UI, you can push them from a local `config.env`:
GitHub CLI (`gh`) is required for this flow, and it must be authenticated (`gh auth login`).

<Steps>
1. **Create `config.env`** anywhere on your machine (see [Configuration](/getting-started/configuration/) for the format).

2. **Set helper defaults in `config.env`** so CLI flags are not required:
   ```bash
   GITHUB_FORK_REPO=<owner/repo>
   SSH_PRIVATE_KEY_PATH=<path/to/private_key>
   ```

3. **Configure your SSH key** — reuse an existing key or generate a new one:
   ```bash
   ssh-keygen -t ed25519 -f ~/.ssh/homerow_ed25519 -N ""
   ```

4. **Push secrets** to your fork without cloning the repo:
   ```bash
   curl -fsSL https://raw.githubusercontent.com/guilhermeprokisch/homerow/main/scripts/fork-deploy.sh | bash -s -- --config ./config.env
   ```

   After secrets are pushed, the script asks whether to trigger workflow `Deploy Mail Server` immediately.

   Example with a dedicated key:
   ```bash
   curl -fsSL https://raw.githubusercontent.com/guilhermeprokisch/homerow/main/scripts/fork-deploy.sh | bash -s -- --config ./config.env --repo <owner/repo> --ssh-key ~/.ssh/homerow_ed25519
   ```

   Example with your default key:
   ```bash
   curl -fsSL https://raw.githubusercontent.com/guilhermeprokisch/homerow/main/scripts/fork-deploy.sh | bash -s -- --config ./config.env --repo <owner/repo> --ssh-key ~/.ssh/id_ed25519
   ```
</Steps>

:::note
On first deploy, the workflow sets up the remote Terraform state bucket, then stores stack state remotely for all future runs.
:::

## SSH Key

:::note
Deploy always needs an SSH key.
For local commands, use `SSH_PRIVATE_KEY_PATH`.
For GitHub Actions, use `SSH_PRIVATE_KEY` secret content.
If you manually add secrets in your fork, set `SSH_PRIVATE_KEY` to the full private key content (not a filesystem path).
:::
