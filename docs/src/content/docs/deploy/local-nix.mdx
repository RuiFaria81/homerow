title: Deploy with Local Nix
description: Deploy directly using a local Nix installation.
---

If you already use Nix (or are on NixOS), you can run deploy directly.

:::note
For most users, the recommended method is [Fork-and-Deploy (GitHub Actions)](/deploy/github-actions/).
Use this local Nix/NixOS path when you want deployment to run from your own machine.
:::

<Steps>
1. **Clone and enter the repository:**
   ```bash
   git clone <repo-url>
   cd homerow
   ```

2. **Create `config.env`** in the repo root. See [Configuration](/getting-started/configuration/) for the full format.

3. **Configure your SSH key** — reuse an existing key or generate one:
   ```bash
   ssh-keygen -t ed25519 -f ~/.ssh/homerow_ed25519 -N ""
   ```
   Then set `SSH_PRIVATE_KEY_PATH` in `config.env`:
   ```bash
   SSH_PRIVATE_KEY_PATH=~/.ssh/homerow_ed25519
   ```

4. **Run deploy:**
   ```bash
   ./hrow deploy --via local
   ```

   If your machine doesn't already have deploy dependencies available, enter a dev shell first:
   ```bash
   nix develop
   ./hrow deploy --via local
   ```
</Steps>

:::note
Deploy always needs an SSH key. For this local method, set `SSH_PRIVATE_KEY_PATH`.
:::

The dev shell (`devenv.nix`) includes deploy tools like `terraform`, `openssh`, and `git`.
