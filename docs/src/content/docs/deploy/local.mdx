Use this page when you want deployment to run from your own machine.

:::note
For most users, the recommended method is [Fork-and-Deploy (GitHub Actions)](/deploy/github-actions/).
:::

## Shared Setup

<Steps>
1. **Clone and enter the repository:**
   ```bash
   git clone <repo-url>
   cd homerow
   ```

2. **Create `config.env`** in the repo root. See [Configuration](/getting-started/configuration/) for the full reference.

3. **Configure your SSH key** — reuse an existing key or generate one:
   ```bash
   ssh-keygen -t ed25519 -f ~/.ssh/homerow_ed25519 -N ""
   ```
   Then set `SSH_PRIVATE_KEY_PATH` in `config.env`:
   ```bash
   SSH_PRIVATE_KEY_PATH=~/.ssh/homerow_ed25519
   ```
</Steps>

## Run Deploy

### Docker / Podman

If Docker or Podman is installed:

```bash
./hrow deploy --via docker
```

### Nix / NixOS

If your machine already has deploy dependencies available:

```bash
./hrow deploy --via local
```

Or enter a dev shell first:

```bash
nix develop
./hrow deploy --via local
```

:::note
Deploy always needs an SSH key. For local deploy methods, set `SSH_PRIVATE_KEY_PATH` in `config.env`.
:::
