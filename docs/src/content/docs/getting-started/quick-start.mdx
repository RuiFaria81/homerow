There are three ways to deploy. **Recommended default: Remote Fork-and-Deploy with GitHub Actions.**
Use the local methods when you specifically want local execution and control.

| Method | Requires | Best for |
|--------|----------|----------|
| [Remote: Fork-and-Deploy (GitHub Actions)](/deploy/github-actions/) | GitHub account | **Recommended** for most users |
| [Local: Docker / Podman](/deploy/docker/) | Docker or Podman | No local Nix installation needed |
| [Local: Nix / NixOS](/deploy/local-nix/) | Nix installed | Most direct, full control |

## Prerequisites

All methods require:

- A domain you control (for DNS records)
- A [Hetzner Cloud](https://www.hetzner.com/cloud) account (VPS provider)
- A [Cloudflare](https://www.cloudflare.com/) account (DNS provider)
- An SSH key pair for server access

## Steps Overview

<Steps>
1. **Create your `config.env`** with domain, credentials, and provider tokens. See [Configuration](/getting-started/configuration/) for all variables.

2. **Generate or reuse an SSH key** for deploy and server access:
   ```bash
   ssh-keygen -t ed25519 -f ~/.ssh/homerow_ed25519 -N ""
   ```

3. **Run deploy** using your chosen method. The deploy will:
   - Set up remote Terraform state
   - Provision VPS, DNS records, and backup storage
   - Bootstrap NixOS on the server
   - Configure mail services, webmail, sync engine, and backups
   - For local methods (Docker/Podman and Nix/NixOS), clone the repository first.

4. **Access your services** after deploy completes:
   - Webmail: `https://webmail.<your-domain>`
   - Rspamd UI: `https://rspamd.<your-domain>`
   - IMAP/SMTP: `mail.<your-domain>`
</Steps>

Choose a deploy method from the sidebar to continue with detailed instructions.
