---
title: Destroy
description: How to tear down your deployment and stop paying for resources.
---

Use destroy when you want to fully tear down this deployment and stop paying for the provisioned resources.

## What Destroy Does

`./hrow destroy` will:

- Destroy Terraform-managed DNS and VPS resources
- Optionally destroy backup data and bucket resources when `--delete-storage` is set (permanent data loss)
- Remove local Terraform state/artifacts
- Remove local deploy keys and generated settings files

## Running Destroy

```bash
./hrow destroy
```

To also delete backup data and bucket resources (permanent data loss):

```bash
./hrow destroy --delete-storage
```

:::danger
This is destructive. After destroy, your server and mailbox data in that environment are no longer available unless you restore from backups.
:::

## Before Destroying

Consider:

- **Verify backups** — ensure your restic backups are up to date if you need to restore data later.
- **Export data** — download any mail or data you want to keep from the webmail interface.
- **DNS propagation** — after destroy, DNS records are removed and mail delivery will stop.
